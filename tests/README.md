# การตรวจสอบคลังสื่อ

เครื่องมือตรวจงานอัตโนมัติ แบ่งเป็นสองชั้น ให้รันก่อน push ทุกครั้ง

## รันยังไง

```bash
npm run check   # ชั้น 1 อย่างเดียว เร็ว ไม่ต้องติดตั้งอะไร
npm test        # ชั้น 1 + ชั้น 2 ทั้งหมด (ครั้งแรกสั่ง npm install ก่อน)
```

## ชั้น 1 — ตรวจโครงสร้าง (`scripts/check-media.mjs`)

ไม่ใช้ AI ไม่ใช้ dependency รันด้วย `node` เปล่า ๆ ได้ทันที ตรวจสิ่งที่เทียบไฟล์ตรง ๆ ได้:

- เกมทุกไฟล์ใน `games/` ต้องมีใน `library.json` (กันเหตุ **เกมหายจากเว็บ** ที่เคยเกิดจริงกับ ภารกิจกู้ของเล่น)
- รายการใน `library.json` ต้องมีไฟล์จริง (กัน 404)
- `url` ต้องตรงกับ `jobId` ของตัวเอง
- ทุกเกมต้องมี `<title>`, `lang="th"`, viewport meta และไม่อ้างอิง `http://`

พร้อมกับ `validate-library.mjs` ที่ตรวจ `library.json` ตาม schema

## ชั้น 2 — ตรวจพฤติกรรมเกม (`tests/`)

ใช้ jsdom เล่นเกมแบบอัตโนมัติ ทุกเทสในนี้กันบั๊กที่**เคยเกิดจริงและแก้ไปแล้ว** ไม่ให้กลับมา:

| ไฟล์ | กันบั๊ก |
|---|---|
| `dragon-game.test.mjs` | กดเริ่มใหม่แล้ว loop เดินซ้อน / คำ Tab ไปไม่ถึง / เล่นด้วยคีย์บอร์ดไม่ได้ |
| `factory-game.test.mjs` | ดาวความพยายามให้ 3 ดวงเสมอไม่ว่าตอบผิดกี่ครั้ง |
| `contrast.test.mjs` | สีปุ่มต่ำกว่าเกณฑ์ WCAG AA (อ่านค่าสีจากไฟล์เกมจริง) |

ทุกเทสถูกพิสูจน์แล้วว่า **ล้มเมื่อรันกับโค้ดก่อนแก้** จึงมั่นใจได้ว่ามันจับ regression จริง ไม่ใช่เทสหลอกที่ผ่านทุกอย่าง

## เพิ่มเกมใหม่แล้วต้องทำอะไร

1. วางไฟล์ไว้ใน `games/`
2. เพิ่มรายการใน `library.json`
3. `npm run check` — ถ้าลืมข้อ 2 จะเตือนทันที
